## 1.Опис предметного середовища

1.1 Предметним середовищем даної роботи є сфера інформаційних веб-ресурсів, орієнтованих на висвітлення новин, оглядів та аналітики у галузі відеоігор.

Сучасна індустрія відеоігор є динамічною та швидкозмінною, що зумовлює постійну потребу користувачів в актуальній інформації щодо нових релізів, оновлень, тенденцій ринку та експертних оцінок ігор. Користувачами таких ресурсів є геймери, аналітики, журналісти та всі зацікавлені особи, які прагнуть отримувати якісний контент українською мовою.

1.2 Бізнес-вимоги 

-Збільшити кількість постійних відвідувачів веб-ресурсу на 20% протягом року за рахунок надання актуального та якісного контенту українською мовою.

1.3 Системні вимоги 

-Система повинна забезпечувати публікацію новин, оглядів та аналітичних матеріалів.

-Система повинна зберігати та відображати інформацію про статті (заголовок, дата, опис, категорія).

-Система повинна підтримувати адаптивний інтерфейс для різних пристроїв.

1.5 Бізнес-логіка системи

Бізнес-логіка веб-застосунку визначає правила обробки та відображення інформації:

-новини та статті додаються до системи та зберігаються;

-користувач переглядає список новин;

-новини сортуються за датою публікації;

-користувач може відкрити повний текст статті;

-кожна стаття належить до певної категорії;

-контент оновлюється адміністрацією ресурсу.

1.6 Функціональні вимоги

Система повинна:

-забезпечувати перегляд списку новин;

-дозволяти відкривати окрему статтю;

-відображати заголовок, короткий опис та дату;

-забезпечувати навігацію по сайту;

-відображати адаптивне меню;

-підтримувати відображення контенту у вигляді карток;

-забезпечувати коректну роботу на різних пристроях.

1.7 Нефункціональні вимоги

Система повинна відповідати таким вимогам:

-час завантаження сторінки не більше 2 секунд;

-адаптивність для різних типів пристроїв (ПК, планшет, смартфон);

-кросбраузерність (Chrome, Firefox, Edge);

-зручність використання (UX/UI);

-стабільність роботи;

-логічна структура коду та файлів.

## 2. Хід виконання

2.1 На початковому етапі розробки було створено кореневу папку проєкту gamehub, всередині якої розміщено головний файл index.html. Для забезпечення зручності підтримки та масштабування проєкту , було створено логічно структуровані підкаталоги:/css — для таблиць стилів;/js — для скриптів;/assets — для графічних ресурсів

![Скрін 3](/assets/labs/lab-1/структура.png)

2.2 Розробка HTML-структури. 

У файлі index.html було розмічено основні структурні компоненти інтерфейсу за допомогою семантичних тегів HTML5:
-Шапка (header), що містить логотип "GameHub" та навігаційне меню.
-Основний блок (main), в якому розміщено інформаційні картки зі статтями.
-Нижній колонтитул (footer) із контактною інформацією.

![Скрін 4](/assets/labs/lab-1/html-структура.png)

2.3 Стилізація та створення адаптивної сітки (CSS).

Для оформлення сторінки було створено файл style.css. Для забезпечення гнучкості інтерфейсу використано відносні одиниці вимірювання (vw, vh, rem, %).
Розташування карток у блоці main реалізовано за допомогою технології CSS Grid (display: grid), що дозволило створити адаптивну сітку, яка автоматично підлаштовується під розмір вікна.

![Скрін 5](/assets/labs/lab-1/css-grid.png)

2.4 Реалізація медіа-запитів та бургер-меню

Для забезпечення адаптивності на пристроях з шириною екрана менше 768px та 480px використано медіа-запити @media. Звичайне меню приховується, сітка стає одноколонковою, і з'являється інтерактивна кнопка-бургер. Логіка плавного відкриття та закриття меню реалізована за допомогою мови JavaScript у файлі script.js та CSS-анімацій (transition).
Також додано візуальні ефекти (hover-ефекти) для інтерактивності карток при наведенні курсора.

## Висновки

У ході виконання лабораторної роботи було успішно спроєктовано та розроблено адаптивний веб-застосунок "GameHub". Використання семантичної HTML-розмітки та сучасних технологій CSS (Grid, Flexbox, медіа-запити) дозволило створити гнучкий інтерфейс, який коректно відображається на десктопних, планшетних та мобільних пристроях. Додатково було реалізовано інтерактивне «бургер-меню» за допомогою JavaScript.

Також було закріплено практичні навички роботи з системою контролю версій Git: налаштовано розгалуження (main, develop, feature-гілки), використано стандарти написання комітів (Conventional Commits) та здійснено успішну публікацію проєкту на платформі GitHub. Розроблений застосунок повністю відповідає поставленим функціональним та стилістичним вимогам.
